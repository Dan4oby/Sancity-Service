Автоматическая установка в powershell

Команда сама создает директорию Sancity на рабочем столе, удаляет предыдущую версию и скачивает обновлятор нужной разрядности

  1. ПКМ по пуску → PowerShell / Терминал

  2. Либо Win+R → powershell → Enter

  3. Вставить команду

[System.Net.ServicePointManager]::SecurityProtocol = [Enum]::ToObject([Net.SecurityProtocolType], 3072); $wc=New-Object Net.WebClient; $wc.DownloadFile('https://raw.githubusercontent.com/Dan4oby/Sancity-Service/refs/heads/main/download_updater.bat', "$env:TEMP\download_updater.bat"); $c=[System.IO.File]::ReadAllText("$env:TEMP\download_updater.bat"); [System.IO.File]::WriteAllText("$env:TEMP\download_updater.bat", $c, [System.Text.Encoding]::Default); &"$env:TEMP\download_updater.bat"

Должно работать ВЕЗДЕ:

bitsadmin /transfer myDownloadJob /download /priority high "https://raw.githubusercontent.com/Dan4oby/Sancity-Service/refs/heads/main/download_updater.bat" "$env:TEMP\download_updater.bat"; $c=[System.IO.File]::ReadAllText("$env:TEMP\download_updater.bat"); [System.IO.File]::WriteAllText("$env:TEMP\download_updater.bat", $c, [System.Text.Encoding]::Default); &"$env:TEMP\download_updater.bat"

FAQ

Пользователь не идентифицирован

Неправильные логин или пароль для входа, проверь внимательно

Невозможно выполнение обновления конфигурации в командном режиме

Вероятно, обновляется нетиповая конфигурация - к сожалению, нужно обновлять вручную

Файл не содержит доступных обновлений

Обновление старее, либо слишком новое для конфигурации, либо скачана вообще другая конфигурация (например, вместо проф - базовая)

компьютер: 1-SRV, пользователь: Остякова Алёна, сеанс: 5, начат: 21.03.2017 в 17:12:28, приложение: Толстый клиент;

Значит, что в базе работает пользователь - нужно запустить заново, заблокировав базу
